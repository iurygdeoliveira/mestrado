##########
# DOCKER #
##########

- Verificar quanto de espaço está sendo ocupado
sudo docker system df
sudo docker system df -v

- Remover todas as imagens
sudo docker system prune -a

- Remover containers, volumes e imagens
docker stop $(docker ps -aq) && docker rm $(docker ps -aq) && docker volume rm $(docker volume ls -qf dangling=true) && docker rmi $(docker images -q) 

Verificar quais imagens estão presentes na máquina
sudo docker images

Remover uma imagem
sudo docker rmi < id imagem > --force

# GIT #
#######

Remover um arquivo grande a arvore do git
git filter-branch -f --tree-filter 'rm -f /path/to/file' HEAD --all

Resolvendo o erro “fatal: refusing to merge unrelated histories” no Git
Para permitir que o Git faça o merge de dois projetos com históricos diferentes, é só passar o parâmetro --allow-unrelated-histories quando for fazer o pull, assim:

git pull origin master --allow-unrelated-histories
